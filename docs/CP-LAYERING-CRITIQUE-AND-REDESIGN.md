## 제1장. 기존 시스템 해부 -- 무엇이 작동하고, 무엇이 부족한가

### 1.1 현재 아키텍처의 본질

현재 시스템은 **단일 캐릭터 분석 파이프라인**이다.

```
FormDataInput(1명) --> buildGeminiPrompt() --> Gemini AI --> ImageAnalysisResult --> ResultPageMain(탭 뷰)
```

`prompt-builder.ts`의 출력 구조를 보면:
- `traits`: 10개 특성 레이더 차트 (sexy, cute, charisma 등)
- `scentCategories`: 6개 향 계열 점수
- `matchingPerfumes[1]`: 정확히 1개의 향수 매칭
- `comparisonAnalysis`: AI 해석 vs 사용자 입력 비교

여기서 "comparison"이라는 단어가 쓰이지만, 이것은 **캐릭터 간 비교가 아니라 AI와 사용자의 시각 차이 비교**다. CP 모드에 필요한 "두 캐릭터의 케미 분석"과는 완전히 다른 개념이다.

### 1.2 결과 페이지의 구조적 한계

`ResultPageMain.tsx`는 다음 구조를 따른다:

```
Header --> 이미지 + TwitterName --> TabNavigation(3탭) --> [분석/향수/비교] --> 하단 액션
```

이것은 **데이터 대시보드** 패턴이다. 사용자가 탭을 클릭하며 정보를 "열람"한다. 여기에는:
- 감정적 흐름(emotional arc)이 없다
- 서사적 긴장(narrative tension)이 없다
- 클라이맥스 순간이 없다
- 사용자가 스스로 발견하는 즐거움이 없다

1인 분석에서는 이 구조가 어느 정도 작동한다. "내 최애의 향수가 뭔지 빨리 알고 싶다"는 욕구에 직접 응답하기 때문이다. 그러나 CP 분석에서 팬이 원하는 것은 "빠른 정보 확인"이 아니다. **"두 캐릭터가 만나는 순간의 서사적 경험"**이다.

### 1.3 톤앤매너의 강점과 한계

현재 톤 규칙(`prompt-builder.ts` 63-73행):
```
"ㄹㅇ", "실화냐", "개쩐다", "갓벽", "핵인싸"
주접 표현: "우리 애", "짱짱이", "심장 저격", "비주얼 테러"
```

이것은 **1인 숭배형 톤**이다. 화자는 캐릭터를 일방적으로 찬양하는 위치에 있다. CP 모드에서는 화자의 위치가 바뀌어야 한다: "찬양자"에서 **"목격자"**로. "이 캐릭터 미쳤다!"가 아니라 "이 둘이 같이 있으면 진짜..."

### 1.4 공유 시스템의 구조적 결함

`ShareCardNew.tsx`는 430x932px 카드 1장을 생성한다. 이 카드에 들어가는 정보:
- 캐릭터 이미지 1장
- 향수 ID, 이름
- 레이더 차트 (10축)
- 향 카테고리 점수 6개
- 노트 3개 (Top/Mid/Base)
- 키워드 3개

이것은 **정보 카드**이지 **감성 카드**가 아니다. 팬이 트위터에 올릴 때 "이거 봐!"라고 말하게 하는 것은 레이더 차트가 아니다. **한 줄의 강렬한 문장**과 **두 캐릭터가 함께 있는 비주얼**이다.

## 제2장. 핵심 비평 -- 팬덤 감성의 관점에서

### 비평 1: "레이어링 팁"은 조향사를 위한 것이지, 팬을 위한 것이 아니다

"왼쪽 손목에 A를 먼저 뿌리고, 5분 후 B를 그 위에 뿌리세요. 비율은 7:3."

이것은 향수 매장 직원의 안내 멘트다. 팬덤 콘텐츠가 아니다.

팬이 원하는 것:
> "A의 시트러스가 먼저 공기를 가르고 나타나. 날카롭고 투명한 첫인상 -- 마치 아침에 창문을 여는 것처럼. 그런데 시간이 지나면 B의 우디함이 아래에서 슬며시 올라와. A를 감싸기 시작해. A가 눈에 보이는 세계라면 B는 그 아래의 뿌리야. 이 CP의 관계가 딱 이거잖아 -- 처음엔 A가 앞장서는 것 같지만, 결국 B가 모든 걸 받쳐주고 있었던 거. 그 순간을 향으로 겪을 수 있다는 게 미친 거 아냐?"

레이어링은 **기법**이 아니라 **관계의 은유**여야 한다.

### 비평 2: 결과 페이지에 "기승전결"이 없다

현재 탭 구조는 영화를 챕터별로 잘라 메뉴에 넣은 것과 같다. "액션 장면 보기", "로맨스 장면 보기", "엔딩 보기" -- 이렇게 영화를 보는 사람은 없다.

CP 분석 결과는 하나의 **서사적 여정**이어야 한다:

| 단계 | 이름 | 경험 | 감정 곡선 |
|------|------|------|-----------|
| 기(起) | 등장 | 각 캐릭터가 따로 소개됨 | 기대감 상승 |
| 승(承) | 탐색 | 각자의 향기 세계가 펼쳐짐 | 호기심 유지 |
| 전(轉) | 만남 | 두 향이 만나는 순간 연출 | 긴장-폭발 |
| 결(結) | 탄생 | "이 CP만의 향" 정체성 확립 | 감동-만족 |

### 비평 3: 공유 가치(Shareability)가 설계에 없다

현재 공유 플로우: 결과 완성 --> 공유 버튼 --> 모달 --> 링크/이미지 선택

이 과정에서 빠진 것: **"뭘 공유할 것인가"에 대한 설계**. 현재는 분석 결과 전체를 이미지 1장으로 압축할 뿐이다.

CP 팬덤 공유의 핵심은 **한 문장의 킬러 카피**다:
- "우리 CP 향기 궁합이 [폭풍 속의 고요]래 ㅋㅋㅋㅋ"
- "A x B = 비 온 뒤 숲길의 향이라는데 너무 맞잖아"

이 한 문장이 트위터에서 바이럴을 만든다. 레이더 차트는 만들지 못한다.

### 비평 4: "궁합이 나쁘다"는 결과가 나오면 안 된다

이것은 절대 법칙이다. 어떤 조합이든 긍정적 해석이 가능해야 한다. 이것은 점술적 프레이밍과 같다 -- 모든 결과가 "당신에게 의미 있는 이야기"가 되어야 한다.

극과 극인 CP --> "반전 케미! 아무도 예상 못한 조합인데 그래서 더 미쳤다!"
비슷한 CP --> "공명하는 영혼! 같은 파장이라 한 번 맞으면 헤어나올 수 없어!"

## 제3장. 재설계 명세 -- 과제별 상세

### 과제 1: 결과 페이지의 스토리텔링 구조 재설계

#### 3.1.1 구조: 탭 --> 스크롤 챕터

기존 `TabNavigation` + `AnimatePresence` 탭 전환을 버리고, **수직 스크롤 내러티브**로 전환한다.

```
[CPResultPage - 스크롤 구조]

CHAPTER 0: 프롤로그
  "두 세계가 만나려 한다..."
  (어둠 속에서 두 캐릭터 실루엣이 서서히 드러남)

CHAPTER 1: 첫 번째 향 -- [캐릭터A 이름]
  A의 이미지
  A의 향수 매칭 결과 (축약형)
  A의 향기를 한 줄로 표현
  "A의 세계는 [시트러스의 날카로움] 으로 시작돼"
  [스크롤 유도 화살표]

CHAPTER 2: 두 번째 향 -- [캐릭터B 이름]
  B의 이미지
  B의 향수 매칭 결과 (축약형)
  B의 향기를 한 줄로 표현
  "B의 세계는 [우디의 깊이] 로 채워져 있어"
  [스크롤 유도 화살표]

CHAPTER 3: 만남의 순간 -- [클라이맥스]
  *** 이것이 핵심 경험 ***
  두 이미지가 가까워지는 애니메이션
  두 색상 팔레트가 합쳐지는 비주얼
  "그리고... 이 둘이 만났어"

  [케미 타입 공개]
  큰 글씨로: "폭풍 속의 고요" (또는 해당 케미 타입명)

  [레이어링 내러티브]
  시간/공간/감정 메타포를 사용한 2-3문단의 스토리

CHAPTER 4: 이 CP만의 향 -- [결]
  CP 고유 향 이름 (AI 생성)
  합쳐진 레이더 차트
  합쳐진 향 카테고리
  레이어링 순서 가이드 (서사적 톤으로)

CHAPTER 5: 공유 & 컬렉션
  공유 카드 프리뷰
  "이 CP를 컬렉션에 추가"
  "다른 CP도 분석해보기"
```

#### 3.1.2 챕터별 공유 단위

각 챕터가 독립적인 공유 카드를 생성할 수 있어야 한다:
- Chapter 1 공유: "A의 향기 카드" (기존 1인 공유와 유사)
- Chapter 2 공유: "B의 향기 카드"
- Chapter 3 공유: "CP 케미 카드" (**가장 바이럴 가치가 높음**)
- Chapter 4 공유: "CP 향 레시피 카드"

#### 3.1.3 클라이맥스 연출 (Chapter 3)

스크롤이 Chapter 3에 도달하는 순간:
1. 화면 배경이 두 캐릭터 색상의 그라데이션으로 전환
2. 짧은 진동(haptic feedback) -- 모바일 전용
3. 두 이미지가 중앙으로 모이는 모션
4. 케미 타입명이 타이핑 효과로 한 글자씩 나타남
5. 레이어링 내러티브가 페이드인으로 서서히 공개

이 순간이 팬이 "와!" 하는 지점이다. 데이터 차트를 보여주는 것이 아니라, **감정적 반응을 유도하는 연출**이다.

### 과제 2: "레이어링" 메타포의 창의적 확장

#### 3.2.1 메타포 분류 체계

레이어링 메타포는 CP의 특성에 따라 동적으로 선택되어야 한다. AI가 두 캐릭터의 traits를 분석한 뒤, 가장 적합한 메타포 축을 2-3개 조합하여 내러티브를 구성한다.

**시간축 메타포 (Temporal)**
관계를 시간의 흐름으로 표현한다.

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| A=freshness 높음, B=darkness 높음 | "A는 새벽의 첫 빛이고 B는 자정의 마지막 그림자야. 이 CP의 하루는 24시간이 다 드라마" |
| A=cute 높음, B=elegance 높음 | "A가 봄의 첫 꽃이면 B는 가을의 마지막 잎. 한 해를 관통하는 향의 서사" |
| 둘 다 energy 높음 | "이 둘의 향은 동시에 터져. 시차 없는 폭발. 같은 순간에 타오르는 두 불꽃" |

**공간축 메타포 (Spatial)**
관계를 장소와 공간으로 표현한다.

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| A=freedom 높음, B=elegance 높음 | "A는 바다 위를 달리는 바람이고 B는 성 안의 정원이야. 바람이 정원에 닿으면 꽃이 흔들려" |
| A=darkness 높음, B=purity 높음 | "A의 그림자가 드리운 골목 끝에 B의 햇살이 비치는 광장이 있어" |

**감정축 메타포 (Emotional)**
관계를 감정의 변화로 표현한다.

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| A=sexy 높음, B=cute 높음 | "A의 향에 심장이 뛰기 시작하면, B의 향이 그 심장을 살살 달래줘. 설렘 다음에 오는 안도감" |
| 둘 다 charisma 높음 | "두 향 다 양보를 안 해. 팽팽한 긴장감이 공기를 채우는데... 그 긴장이 이 CP의 매력이야" |

**음악 메타포 (Musical)**
관계를 음악적 요소로 표현한다.

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| 상호보완적 traits | "A가 멜로디를 시작하면 B가 화음으로 응답해. 혼자 들으면 좋은 곡인데, 같이 들으면 소름" |
| 극과 극 traits | "A가 록이면 B는 클래식. 매시업하면 미친다는 걸 누가 알았겠어" |
| 비슷한 traits | "같은 키로 노래하는 두 사람. 유니즌 그 자체. 틀릴 수가 없는 화음" |

**색채 메타포 (Chromatic)**
두 캐릭터의 personalColor를 활용한다.

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| A=warm tone, B=cool tone | "따뜻한 주황과 차가운 파랑이 만나면 일몰의 하늘이 돼. 이 CP의 색은 황혼" |
| 비슷한 palette | "같은 물감통에서 나온 두 색. 번지면 경계가 사라지는 수채화" |

**요리 메타포 (Culinary)**

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| A=sweet 계열, B=spicy 계열 | "A가 달달한 마카롱이면 B는 매운 깐풍기. 단짠단짠의 정석 CP" |
| 한쪽이 dominant | "A가 메인 디쉬면 B는 완벽한 소스. B 없는 A는 뭔가 2% 부족해" |

**자연 현상 메타포 (Natural Phenomena)**

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| 극적 대비 | "A는 태양이고 B는 달이야. 평소엔 절대 같이 안 뜨는데, 일식 순간 겹치면... 세상이 멈춰" |
| 유사성 높음 | "같은 바다에서 일어나는 두 파도. 부딪히면 더 높아지고 더 멀리 가" |
| 한쪽이 감싸는 관계 | "B는 비고 A는 땅이야. B가 내리면 A에서 풀이 자라. A 없는 B는 그냥 물이고, B 없는 A는 마른 땅이야" |

**텍스타일 메타포 (Fabric/Texture)**

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| A=luxury 높음, B=purity 높음 | "A의 벨벳 위에 B의 린넨이 올라가면? 무거운 고급스러움 위의 가벼운 순수함. 이 질감의 대비가 이 CP" |

**건축 메타포 (Architectural)**

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| 구조적 보완 관계 | "A가 기둥이면 B는 지붕. 따로 있으면 그냥 건축 자재인데, 합치면 비를 막아주는 집이 돼" |

**게임/전투 시너지 메타포 (Combat Synergy)**

| 적용 조건 | 메타포 예시 |
|-----------|-------------|
| 공격/수비 대비 | "A가 어그로를 끌면 B가 뒤에서 딜을 꽂아. 이 CP의 콤보 데미지는 측정 불가" |

#### 3.2.2 메타포 조합 원칙

AI가 메타포를 생성할 때:
1. **주 메타포 1개**: CP의 핵심 관계성을 대표하는 메타포 (가장 길고 서사적)
2. **보조 메타포 1-2개**: 주 메타포를 보완하는 짧은 메타포
3. **절대 같은 축에서 2개를 쓰지 않을 것**: 시간 메타포 2개는 반복이 됨
4. **캐릭터 배경 지식이 있으면**: 해당 작품의 세계관에 맞는 메타포 우선 선택

### 과제 3: CP 전용 톤앤매너 프레임워크

#### 3.3.1 화자 포지션의 전환

| 구분 | 1인 분석 | CP 분석 |
|------|----------|---------|
| 화자 역할 | 숭배자 | 목격자/해설자 |
| 핵심 감정 | "이 캐릭터 미쳤다!" | "이 둘이 같이 있으면 미쳤다!" |
| 시선 방향 | 화자 --> 캐릭터 (일방향) | 화자 --> 캐릭터A <--> 캐릭터B (관찰) |
| 주접 대상 | 캐릭터 자체 | 두 캐릭터 사이의 케미 |

#### 3.3.2 CP 전용 어휘 사전

**케미 표현**:
- "케미 폭발", "시너지 미쳤다", "합이 개쩐다"
- "이 조합 누가 만들었냐", "둘이 같이 있으면 세계관 완성"
- "이 CP 불법 아니야?", "케미 과다로 신고감"
- "원작자가 이걸 의도했을 리가 없는데... 천재인가?"

**향 상호작용 표현**:
- "향이 대화한다" / "향으로 밀당 중"
- "A향이 먼저 말 걸어... B향이 응답했어"
- "두 향 사이에 텐션이 있어"
- "이건 향으로 쓴 러브레터야"
- "A향이 B향을 끌어당기고 있어"
- "두 향이 서로를 완성해"

**관계 동사**:
- "감싸다", "스며들다", "끌어당기다", "받쳐주다"
- "밀어내다(가 결국 끌어안다)", "뒤따르다", "앞서가다"
- "공명하다", "충돌하다(가 폭발적 조화를 만들다)"

**절대 금지 표현**:
- "궁합이 안 맞다", "어울리지 않는다", "별로다"
- "이상한 조합", "안 맞는 향"
- 부정적 결론 자체가 불가능하도록 프롬프트 설계

#### 3.3.3 케미 유형별 톤 분기

| 케미 유형 | 톤 특성 | 예시 문구 |
|-----------|---------|-----------|
| 폭발형(Explosive) | 격앙, 감탄, 충격 | "이거 첫 만남부터 터졌어!!! 공기가 달라졌다고!!!" |
| 슬로우번(Slow Burn) | 감성적, 여운, 서정 | "처음엔 몰랐는데... 시간이 지나니까... 아... 이거였구나..." |
| 밀당형(Push-Pull) | 긴장감, 장난기, 밀고 당기기 | "A가 다가가면 B가 한 발 빼고, B가 돌아서면 A가 잡아당기고... 이 향의 텐션 뭐냐" |
| 공명형(Resonance) | 경외, 운명적, 잔잔한 감동 | "이건 찾은 거야. 같은 주파수. 같은 파장. 운명이라는 단어 쓰기 싫었는데 이거 진짜 운명이야..." |
| 반전형(Plot Twist) | 놀라움, 반전, 뜻밖 | "ㅋㅋㅋ 이거 누가 봐도 안 맞을 줄 알았는데?? 근데 왜 이렇게 좋냐?? 반전이다 반전" |
| 보호형(Shield) | 따뜻함, 포근, 안도 | "B의 향이 A를 완전히 감싸고 있어... 이 세상에서 가장 안전한 향기의 담요..." |
| 중력형(Gravity) | 끌림, 저항 불가, 몰입 | "A의 향에 한 번 빠지면 B가 끌어당겨. 이 중력장에서 벗어날 수 없어" |
| 합체형(Fusion) | 경이, 새로운 탄생, 초월 | "A도 아니고 B도 아닌 완전히 새로운 향이 탄생했어. 이건 이 CP만의 우주야" |

#### 3.3.4 "나쁜 궁합"이 불가능한 프레이밍 전략

모든 결과가 긍정적이 되도록 하는 핵심 로직:

1. **traits 차이가 클 때**: "극과 극의 매력! 이렇게 다른데 그래서 더 끌리는 거야!"
2. **traits가 비슷할 때**: "영혼의 쌍둥이! 이런 공명은 우연이 아니야!"
3. **한쪽이 압도적일 때**: "확실한 리드! 근데 리드하는 쪽도 상대 없이는 빛나지 못해!"
4. **둘 다 특징이 약할 때**: "은은한 하모니... 가장 자연스럽고 편안한 케미야"

### 과제 4: 오프라인 행사 경험 설계

#### 3.4.1 현재 플로우의 시간 분석

```
[현재 1인 분석 플로우]
이미지 업로드: ~30초
폼 작성 (스타일/성격/매력): ~60-90초
AI 분석 대기: ~15-30초
결과 확인: ~180-300초
---
총 소요: 5-8분
```

행사에서 한 부스당 3-5분 제한이라면, CP 분석(2명)은 절대 현재 플로우로 불가능하다.

#### 3.4.2 이벤트 최적화 플로우 설계

**Option A: "빠른 CP 분석" (2인 동시 참여)**

```
[2인 동시 모드 - 소요: 3분]

Person A & B 각각 QR 코드 스캔 (동시)     --> 0초
각자 캐릭터 이미지 1장 업로드 (동시)       --> 30초
각자 핵심 키워드 3개만 선택 (동시)         --> 30초
"매칭 시작!" 버튼 (한 쪽이 누름)          --> 0초
AI 분석 (두 캐릭터 동시 분석)             --> 20초
"3... 2... 1... 공개!" 카운트다운          --> 5초
양쪽 폰에 동시에 히어로 카드 표시          --> 즉시
---
총 소요: ~90초 + 결과 감상 90초 = 3분
```

**Option B: "솔로 CP 분석" (1인 2캐릭터)**

```
[1인 모드 - 소요: 3분]

QR 코드 스캔                              --> 0초
캐릭터A 이미지 업로드                      --> 15초
캐릭터B 이미지 업로드                      --> 15초
(폼 생략 -- AI가 이미지만으로 분석)        --> 0초
AI 분석                                   --> 20초
히어로 카드 표시                           --> 즉시
"전체 결과는 QR로 나중에 보기" 안내         --> ---
---
총 소요: ~50초 + 결과 감상 100초 = ~3분
```

#### 3.4.3 이벤트 전용 "히어로 카드" 설계

전체 스크롤 내러티브가 아닌, 핵심만 담은 한 화면:

```
+------------------------------------------+
|                                          |
|  [캐릭터A 이미지]  x  [캐릭터B 이미지]    |
|                                          |
|        --- CP CHEMISTRY TYPE ---         |
|           "폭풍 속의 고요"                 |
|                                          |
|  A의 향: AC'SCENT 07    B의 향: AC'SCENT 14 |
|                                          |
|  "A의 시트러스가 B의 우디를 만나면          |
|   비 온 뒤 숲길 같은 향이 돼.              |
|   이 CP의 향은 [정적 속의 폭풍]이야"       |
|                                          |
|  [QR: 전체 결과 보기]  [공유하기]          |
+------------------------------------------+
```

2초 안에 핵심 정보가 전달되어야 한다:
1. 어떤 두 캐릭터인지 (이미지)
2. 케미 타입 (큰 글씨)
3. 한 줄 킬러 카피 (감성 문구)

#### 3.4.4 물리적 향수 체험 연동

부스에서의 경험 시나리오:

```
1. 디지털 분석 결과 확인 (히어로 카드)
2. "직접 맡아볼까요?" --> 부스 직원이 안내
3. A의 향수를 무향지에 뿌려서 전달
4. "A의 향을 먼저 맡아보세요... 어때요?"
5. B의 향수를 같은 무향지에 겹쳐 뿌림
6. "이제 B의 향이 합류했어요. 달라졌죠?"
7. "이게 이 CP의 향이에요" --> "와...!" 반응
8. 선택: 실제 구매 or 사진 촬영
```

이 순간이 디지털 분석의 물리적 확인이 되며, 구매 전환의 핵심 접점이다.

### 과제 5: 반복 사용 동기 설계

#### 3.5.1 CP 도감(Codex) 시스템

```
[마이페이지 - CP 도감]

나의 CP 컬렉션 (3/∞)
+-------+  +-------+  +-------+
| A x B |  | C x D |  | E x F |
| 폭발형 |  | 공명형 |  | 밀당형 |
|  ✨🔥  |  |  💫✨  |  |  💕⚡  |
+-------+  +-------+  +-------+

케미 타입 도감 진행률: 3/8 (37.5%)
[ ] 슬로우번  [ ] 반전형  [ ] 보호형
[ ] 중력형    [ ] 합체형
```

수집 요소:
- **케미 타입 컴플리트**: 8가지 케미 타입을 모두 모으면 특별 배지
- **작품별 CP 맵**: 한 작품의 캐릭터들로 만들 수 있는 모든 CP 조합
- **향 계열 다양성**: 시트러스+우디, 플로럴+머스크 등 다양한 향 조합 경험

#### 3.5.2 같은 CP의 변주

같은 캐릭터 조합이라도 다른 결과를 유도하는 방법:

1. **설정 분기**: "원작 설정"과 "if 설정"에서 다른 키워드를 선택하면 다른 결과
2. **계절 한정 분석**: 봄/여름/가을/겨울 각각의 레이어링 결과가 다름
3. **감정 상태 입력**: "행복한 날의 이 CP" vs "힘든 날의 이 CP"
4. **시점 선택**: "첫 만남의 향" vs "오래된 관계의 향" vs "이별 직전의 향"

#### 3.5.3 커뮤니티 & 소셜 기능

- **인기 CP 랭킹**: "이번 주 가장 많이 분석된 CP TOP 10"
- **케미 타입 통계**: "이 CP를 분석한 87%가 [슬로우번] 결과를 받았어요"
- **익명 공유 갤러리**: 다른 팬들의 CP 분석 결과를 구경
- **"도전" 시스템**: "이번 주 챌린지: 같은 작품에서 3개 이상 CP 분석하기"

#### 3.5.4 시즌 & 이벤트 특별 분석

| 이벤트 | 특별 기능 | 기간 |
|--------|-----------|------|
| 밸런타인 | "로맨스 강화" 분석 -- 감정축 메타포 가중 | 2월 한정 |
| 할로윈 | "다크 케미" 분석 -- darkness 기반 서사 강화 | 10월 한정 |
| 크리스마스 | "겨울 레이어링" -- 따뜻함/포근함 메타포 특화 | 12월 한정 |
| 화이트데이 | "역방향 분석" -- B-->A 관점에서 다시 분석 | 3월 한정 |
| 코미켓/서코 | "이벤트 모드" -- 빠른 분석 + 부스 연동 | 행사 기간 |

## 제4장. 기술 구현 방향 (개요)

### 4.1 새로운 타입 정의

```typescript
// CP 분석 결과 타입 (개념)
interface CPAnalysisResult {
  characterA: {
    name: string
    image: string
    analysis: ImageAnalysisResult  // 기존 타입 재사용
  }
  characterB: {
    name: string
    image: string
    analysis: ImageAnalysisResult
  }
  chemistry: {
    type: ChemistryType  // 'explosive' | 'slow_burn' | 'push_pull' | ...
    typeName: string     // "폭풍 속의 고요" 같은 고유 이름
    score: number        // 0.85-1.0 (항상 높음)
    layeringNarrative: {
      primaryMetaphor: string    // 주 메타포 (3-5문장)
      secondaryMetaphors: string[]  // 보조 메타포 (각 1-2문장)
      metaphorTypes: MetaphorType[] // 사용된 메타포 축
    }
    layeringGuide: {
      order: 'A_first' | 'B_first' | 'simultaneous'
      narrativeReason: string  // 서사적 이유 설명
    }
    cpScentName: string       // "비 온 뒤 숲길" 같은 CP 고유 향 이름
    killerCopy: string        // 공유용 한 줄 카피
    detailedStory: string     // 5-7문장의 상세 케미 스토리
  }
}

type ChemistryType =
  | 'explosive'    // 폭발형
  | 'slow_burn'    // 슬로우번
  | 'push_pull'    // 밀당형
  | 'resonance'    // 공명형
  | 'plot_twist'   // 반전형
  | 'shield'       // 보호형
  | 'gravity'      // 중력형
  | 'fusion'       // 합체형

type MetaphorType =
  | 'temporal'     // 시간축
  | 'spatial'      // 공간축
  | 'emotional'    // 감정축
  | 'musical'      // 음악
  | 'chromatic'    // 색채
  | 'culinary'     // 요리
  | 'natural'      // 자연 현상
  | 'textile'      // 텍스타일
  | 'architectural' // 건축
  | 'combat'       // 게임/전투
```

### 4.2 새로운 프롬프트 빌더

`buildCPGeminiPrompt(characterA, characterB, locale)` 함수가 필요하다.

기존 `buildGeminiPrompt`의 톤 규칙을 계승하되:
- 화자 포지션 변경 (숭배자 --> 목격자)
- CP 전용 어휘 사전 추가
- 메타포 생성 지침 추가
- 부정적 결론 불가 제약 추가
- 케미 타입 분류 기준 추가

### 4.3 새로운 결과 페이지 컴포넌트

```
src/app/[locale]/result/components/cp/
  CPResultPage.tsx        -- 메인 스크롤 내러티브
  CPChapterPrologue.tsx   -- Chapter 0: 프롤로그
  CPChapterCharacter.tsx  -- Chapter 1-2: 개별 캐릭터
  CPChapterClimax.tsx     -- Chapter 3: 만남의 순간
  CPChapterResolution.tsx -- Chapter 4: CP의 향
  CPShareCard.tsx         -- CP 전용 공유 카드
  CPHeroCard.tsx          -- 이벤트용 히어로 카드
  CPChemistryBadge.tsx    -- 케미 타입 배지
```

### 4.4 기존 시스템과의 호환

CP 모드는 기존 1인 분석을 **대체하는 것이 아니라 확장**하는 것이다.

```
메인 화면
  |-- [캐릭터 향수 분석] --> 기존 1인 플로우 (변경 없음)
  |-- [CP 케미 분석] --> 새로운 CP 플로우
```

기존 `ImageAnalysisResult` 타입은 CP 분석의 하위 구성요소로 재사용된다. 기존 컴포넌트(TraitRadarChart, PerfumeNotes 등)도 CP 챕터 내에서 재사용 가능하다.

|   [A 이미지]     x     [B 이미지]         |
